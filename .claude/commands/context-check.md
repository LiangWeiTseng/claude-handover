# /context-check — Context 健康度分析

## 使用時機
- 每 3–4 個 session 做一次例行檢查
- 當你覺得回應速度變慢、或 Claude 開始草草結束任務
- 在開始一個新的大任務之前

## 執行步驟

執行以下分析，用**繁體中文**回覆：

### Step 1：掃描記憶索引
讀取 `MEMORY.md`，列出所有記憶檔及其描述。

### Step 2：對每個記憶檔評估相關性
對每個記憶項目，判斷：
- **保留**：對當前工作任務直接有用
- **可壓縮**：資訊是對的但太冗長，可以縮短
- **雜訊**：目前用不到，或已被更新的資訊取代

### Step 3：輸出報告
格式如下：

```
## Context 分析報告

### 雜訊（建議移除或壓縮）
- [檔名] — 原因

### 核心保留
- [檔名] — 為什麼需要

### 建議行動
1. ...
2. ...

### 目前核心任務進度（3 行以內）
...
```

### Step 4：詢問是否執行壓縮
報告完後問使用者：「要現在執行壓縮嗎？」
若回答是，才進行刪除或改寫記憶檔。

## 注意事項
- 不要在沒問過使用者的情況下自行刪除記憶
- 判斷「相關性」要以**當前工作目錄的任務**為基準
- 輸出要簡潔，總字數控制在 300 字以內
