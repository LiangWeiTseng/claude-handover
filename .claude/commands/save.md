# /save — Session 收尾（context 檢查 + 交班）

## 使用時機
記不住有哪些指令？用 /save 就對了。
適合：切換任務前、session 結束前、要 /clear 之前。

## 執行順序

用**繁體中文**依序執行：

---

### PART 1：Context 健康度快速檢查

讀取當前 session 的記憶索引（MEMORY.md），對每個記憶檔快速判斷：
- 本次 session 用到了嗎？
- 是否有過時或重複的內容？

輸出一行摘要：「記憶狀態：X 個保留 / Y 個可壓縮 / Z 個雜訊」

若有雜訊，列出是哪些，**但不要自動刪除**——等 PART 3 一起問。

---

### PART 2：交班單

從當前對話提取，格式如下：

```
## 交班單（[今天日期]）

### 這次完成了（最多 4 點）
- ...

### 關鍵決策（為什麼這樣做）
- ...

### 還沒解決 / 下次要繼續
- ...

### 下一步（具體可執行）
1. ...
```

---

### PART 3：詢問行動

輸出以下選項，讓使用者選：

```
收尾選項：
[A] 更新交接檔並準備 /clear（推薦：切換任務）
[B] 只更新交接檔，繼續當前 session
[C] 同時壓縮雜訊記憶 + 更新交接檔 + 準備 /clear

選哪個？（輸入 A / B / C）
```

根據回答執行對應操作。

**更新交接檔時：**
1. 先讀專案根目錄的 `CLAUDE.md`，找「Session 結束」段落——它會指定本專案要更新哪些交接檔，以及各自是 append 還是覆寫。
2. 依該段落指定的檔案與規則更新。
3. 若專案沒有 `CLAUDE.md` 或未指定交接檔，退回預設：找工作目錄下的 `WORKLOG.md`（通常在專案根或 `.claude/projects/.../memory/`），補本次 delta。

---

## 注意事項
- 交班單優先寫「為什麼」，不是「做了什麼」
- 交接檔的更新規則由專案 `CLAUDE.md` 定義，skill 只負責執行；append 類的檔只補 delta，不整份重寫
- 若記憶狀態乾淨（無雜訊），PART 1 只輸出一行就跳過
- 總時間控制：這個 skill 應在 2-3 次對話內完成
