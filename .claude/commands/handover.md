# /handover — 交班 + 準備 /clear

## 使用時機
- 切換到不同任務之前
- Session 要結束前
- 要用 /clear 清 context 之前
- 換裝置 / 換 AI 工具前

## 執行步驟

用**繁體中文**執行以下操作：

### Step 1：產生交班摘要
從當前對話，提取：

```
## 交班單（[今天日期]）

### 這次做了什麼（最多 5 點）
- ...

### 關鍵決策
- [決策內容] → [原因]

### 卡住的地方 / 未解決
- ...

### 下一步（直接可以做的）
1. ...
2. ...

### 注意事項（下次 session 要知道的）
- ...
```

### Step 2：更新 WORKLOG.md
找到當前工作目錄下的 `WORKLOG.md`（通常在專案根目錄或 `.claude/projects/.../memory/`）。

在對應區塊**補充**本次更新內容。不要整份重寫，只更新有變動的部分。

### Step 3：詢問是否同步 memory
問：「要同步更新 .claude memory 資料夾裡的相關記憶檔嗎？」
若回答是，則一併更新對應的 project memory 檔案。

### Step 4：建議下一步
輸出：
```
✓ 交班完成。

建議：
- 輸入 /clear 清除當前 context，下次 session 從乾淨狀態開始
- 或繼續當前任務（context 還夠）
```

## 注意事項
- 交班單優先記「為什麼這樣做」而不是「做了什麼」
- 「卡住的地方」是最重要的，下次接手的人（或 AI）最需要這個
- 不要重複已在 WORKLOG 裡的完整歷史，只記本次 delta
- 總字數控制在 400 字以內
